Distributed Bitcoin Miner

Written in GoLang, 1000+ lines of code

Skills: Concurrency, Distributed Systems, Load-Balancing, Scheduling.

Built using Live Sequence Protocol, an internet protocol library I wrote to reliably transmit data between clients and servers.

  • Clients will request the server, to compute the minimum hash value of a given string with an integer appended to the end, simulating the compute-intensive task of mining bitcoin.
  • The server, upon recieving requests distributes the work amongst a network of miners.
  • Each miner, uses brute force to compute all hash values for integers within a given range and returns it to the server.
  • Through distribution the time required to handle a request decreases exponentially as the network of miners grow.
  • The server handles multiple requests at a given time through the use of load balancing algorithms, to ensure fairness.
  • The server, client, and miner processes are all fault-tolerant and handle dropped connections appropriately.
